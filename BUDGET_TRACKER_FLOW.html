PocketFlow Budget Tracker — Project Flow

Visual flow from first open through rules, cutoffs, tracking, and optional extras. Based on BUDGET_TRACKER_IMPLEMENTATION_GUIDE.md. All data stays local (Dexie + IndexedDB).

End-to-end flow

First-time profile setup

Shown only when onboarding is not completed. After finish, user enters the main app.

Onboarding

4-step wizard

  • Display Name (required, min 2 chars)
  • Profile Image (optional)
  • PIN Code (5 digits)
  • Face ID toggle (optional)
Storage

Save to Dexie

  • userProfiles table
  • Mark onboarding completed
  • Route to Dashboard / Tracker

Return visit — unlock

If lock is enabled, user unlocks with PIN and/or Face ID before using the app. Profile can be edited later on the Me page.

Lock screen

Unlock options

  • 5-digit PIN
  • Face ID (if enabled)
Me page

Edit anytime

  • Display name
  • Profile image
  • PIN / Face ID settings

Enter / edit budget rules

Baseline split is fixed names with editable percentages. Total must equal 100% before save.

RuleDistributionCard

Default rules

  • Expenses — 50%
  • Savings — 30%
  • Wants — 20%
  • Names are fixed; percents editable
  • Validate: total = 100
RulePieChartCard

Visual check

  • Pie chart from current rules
  • Labels: Expenses, Savings, Wants
  • Values: current percentages
Storage

rules table

  • id, name, percent, itemCount
  • Used by every later allocation step

Enter cutoff income (2 per month)

Exactly two cutoff slots per month/cycle. User enters amounts; system auto-distributes by current rules.

CutoffInputCard

User inputs

  • Slot 1 label (default: 1st cutoff)
  • Slot 1 amount
  • Slot 2 label (default: 2nd cutoff)
  • Slot 2 amount
  • Labels editable (e.g. 15, 30)
On save

Auto distribution

  • Store in cycleCutoffs
  • Run distributeByRules(amount, rules)
  • Each cutoff → Expenses / Savings / Wants
cutoffAmount × rule.percent / 100 → rule allocation
Flexible adjustment (optional)

User may manually tweak rule amounts inside a cutoff. Validate: sum(ruleAmountsForCutoff) ≤ cutoffAmount. If exceeded: “Total rule amount cannot exceed cutoff amount.” Store line items in budgetEntries.

Build reusable items

Create reusable items under Expenses, Savings, and Wants. Active items also populate dropdown options when adding budget entries.

ItemBuilderCard

Item builder

  • Item name
  • Amount (optional, default 0)
  • Categories (multi-select)
  • Active toggle
  • Have child items toggle
  • Save to itemBuilders
  • Dropdown options when adding entries under a rule
  • New names saved to itemBuilders
Child items

If children enabled

  • Parent progress bar
  • Add child: name + amount
  • Save to itemBuilderChildren
remaining = parentAmount − sum(children)
used% = sum(children) / parentAmount × 100
(if parentAmount = 0 → 0%)

Track spending

Live progress against allocated budgets for the month.

WantsTrackerCard

Wants tracker

  • Allocated Wants for month
  • Used = sum of wants entries
  • Progress bar / % used
  • List: name, amount, date
  • Highlight when used ≥ allocated
BudgetTrackerCard

Overall budget

  • Total allocated
  • Total used
  • Remaining
  • Percent used + main progress bar
≥ 80% → almost runs out (warning)
≥ 100% → exceeded (danger)

Overrun & optional extras

Unexpected expenses appear only when over budget. Others is an optional side budget.

If used > allocated (any expense category)

  • Show UnexpectedExpensesCard
  • overrun = allocated − used (negative)
  • Store in unexpectedExpenses
  • Fields: expense name, amount (e.g. −1000)

If within budget

Unexpected Expenses card stays hidden.

OthersBudgetCard

Optional “Others” budget

  • Optional: Budget Allocated
  • Log expenses: name + amount
  • othersBudgets + othersExpenses
  • Own progress bar

How inputs become allocations and trackers (left → right).

Typical user path (happy path)

1 → 2 → 3 → 4

  • Complete onboarding (or unlock)
  • Confirm / edit rules (50/30/20)
  • Enter both cutoff amounts for the month
  • System allocates Expenses, Savings, Wants

5 → 6

  • Add templates and item builders as needed
  • Log wants / expenses against allocations
  • Watch Wants + overall Budget progress

7 (only if needed)

  • Overrun → Unexpected Expenses appears
  • Optional Others budget for side spending
